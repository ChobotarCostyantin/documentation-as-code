# Infrastructure Specification Document (ISD)

## 1. Середовище розгортання
Система розгортається у контейнеризованому середовищі за допомогою інструменту Docker. Це забезпечує ідентичність середовищ розробки (Development) та продакшену (Production).

## 2. Основні інфраструктурні компоненти
Конфігурація `docker-compose` включає три основні сервіси:
- `frontend-app`: Контейнер з Node.js, що обслуговує Next.js застосунок (порт 3000).
- `backend-api`: Контейнер з Node.js, що виконує скомпільований NestJS код (порт 3001).
- `postgres-db`: Офіційний образ PostgreSQL для зберігання персистентних даних (порт 5432).

## 3. Принципи масштабування
На початковому етапі використовується вертикальне масштабування (збільшення CPU/RAM хост-сервера). Завдяки відсутності стану (stateless) у JWT-авторизації, архітектура готова до горизонтального масштабування бекенду - розгортання кількох реплік `backend-api` за балансувальником навантаження (наприклад, Nginx або Traefik).
