# Software Design Document (SDD)

## 1. Загальна архітектура
Система побудована за клієнт-серверною архітектурою. Архітектурний стиль бекенду - модульний моноліт (всі сервіси працюють в одному процесі, але жорстко розділені на логічні домени).

## 2. Основні компоненти
- **Frontend Client (Next.js / React):** Компонент, що відповідає за презентаційний шар. Включає публічний каталог, сторінки профілів та закриту адміністративну панель. Взаємодіє з сервером через HTTP/REST.
- **Backend API (NestJS):** Центральний вузол бізнес-логіки. Складається з ізольованих модулів:
  - `UsersModule` (реєстрація, профілі).
  - `AuthModule` (JWT стратегії, логін).
  - `SoftwareModule` (CRUD операції для софту).
  - `ReviewsModule` (обробка оцінок).
- **Database (PostgreSQL):** Сховище даних, взаємодія з яким відбувається через ORM (TypeORM).

## 3. Взаємодія компонентів
Клієнт ініціює REST API запит (наприклад, `GET /api/software?category=frontend`). Backend API приймає запит, валідує параметри (через DTO), звертається до PostgreSQL через відповідний Service/Repository, отримує дані, серіалізує їх та повертає клієнту у форматі JSON.
