# System Specification Document (SSD)

## 1. Обмеження та припущення
- **Припущення:** Користувачі мають базову технічну грамотність.
- **Обмеження:** Редагування та створення контенту (відгуків) можливе лише після реєстрації. Дані зберігаються виключно в реляційній базі даних (PostgreSQL). Клієнтська частина вимагає браузера з увімкненим JavaScript.

## 2. Основні функції (Межі системи)
- **FR-01:** Управління доступом. Авторизація за допомогою JWT-токенів, управління профілями, розмежування прав (Admin, User).
- **FR-02:** Робота з каталогом. Пошук програмного забезпечення, фільтрація за категоріями, перегляд сторінок інструментів з детальними специфікаціями та скріншотами.
- **FR-03:** Взаємодія з контентом. Можливість для авторизованих користувачів створювати відгуки, оцінювати ПЗ за специфічними метриками та додавати технології у "свій стек".

## 3. Нефункціональні вимоги
- **Продуктивність:** Серверна частина повинна обробляти пошукові запити до бази даних з часом відповіді не більше 300 мс. Фронтенд використовує SSR (Server-Side Rendering) для пришвидшення Time-to-Interactive.
- **Доступність:** Цільовий показник uptime - 99.9%. Система підтримує швидке перезавантаження завдяки контейнеризації.
- **Безпека:** Всі паролі повинні хешуватися за допомогою алгоритму bcrypt. Мутації даних (POST/PATCH/DELETE) захищені Guards, а адмін-маршрути вимагають відповідного Role-Based Access Control (RBAC).
