# Test Strategy

Стратегія забезпечення якості фокусується на тестуванні API та UI.

- **Backend:** Використовуються Unit-тести для перевірки складної бізнес-логіки в NestJS-сервісах. E2E-тести застосовуються для перевірки цілісності REST API ендпоінтів (включаючи взаємодію з тестовою БД).
- **Frontend:** Мануальне кросбраузерне тестування ключових UI-компонентів та автоматизовані інтеграційні тести критичних шляхів користувача (наприклад, процес реєстрації та додавання відгуку).
